### **INCIDENTE: Actualización de URL del Servicio de Usuarios (USERS)**

**1. RESUMEN DE LA TAREA:**
   - Se recibió una solicitud para confirmar y, si era necesario, actualizar la URL del microservicio `USERS`.
   - Nueva URL proporcionada: `https://script.google.com/macros/s/AKfycbxh-kdkom9vq6cHTBe53EMzuIO7qcW6DJO-FkttQCn3ugPO935BQSWnB88qTkLQPNmn/exec`

**2. ANÁLISIS DE ESTADO INICIAL:**
   - **Auditoría de `api.js`:** Se encontró una URL desactualizada para el endpoint `USERS`.
   - **Auditoría de `api-manager.js`:** Se encontró una URL diferente y también desactualizada para el endpoint `USERS`.
   - **Conclusión:** Existía una inconsistencia de URLs para el mismo servicio entre los archivos de configuración de la API, y ninguna de las URLs coincidía con la nueva URL proporcionada, que se considera la fuente de verdad.

**3. SOLUCIÓN IMPLEMENTADA:**
   - Se actualizó la constante `API_ENDPOINTS.USERS` en **ambos archivos (`api.js` y `api-manager.js`)** con la nueva URL proporcionada por el Project Manager.
   - Esta acción centraliza y estandariza el endpoint del servicio `USERS`, resolviendo la inconsistencia y asegurando que todas las llamadas futuras se dirijan a la implementación correcta del servicio.

**4. ESTADO:**
   - **COMPLETADO:** La URL del servicio `USERS` ha sido actualizada y unificada en toda la base de código.
