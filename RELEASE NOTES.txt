# NOTAS DE LANZAMIENTO (RELEASE NOTES) - GPSpedia v2.0

## RESUMEN EJECUTIVO
GPSpedia v2.0 marca un hito en la evolución de la plataforma, transformándose de una herramienta monolítica heredada (v1.5) a una infraestructura técnica de alto rendimiento. Esta versión introduce una arquitectura de microservicios, seguridad avanzada de grado profesional, optimización extrema para dispositivos móviles y un enfoque colaborativo en la calidad de los datos técnicos.

---

## 1. ARQUITECTURA Y NÚCLEO (BACKEND)
- **Ecosistema de Microservicios:** Desacoplamiento total en servicios independientes ruteados dinámicamente (`Auth`, `Catalog`, `Feedback`, `Users`, `Write`, `Utilities`, `Image`).
- **Ruteo Centralizado:** Implementación de `api-config.js` como fuente única de verdad para la comunicación con microservicios de Apps Script.
- **Gestión de Sesiones Segura:** Sistema de tokens de sesión con validación asíncrona mediante un mecanismo de "bloqueo de validación" para prevenir colisiones entre pestañas.
- **Manejo de Errores Dinámico:** Interfaz `ui.js` capaz de exponer errores específicos del backend, facilitando el diagnóstico rápido en campo.

## 2. SEGURIDAD Y PROTECCIÓN DE DATOS (ZONAS SEGURAS)
- **Control de Acceso (RBAC):** Implementación de jerarquía estricta: `Desarrollador` > `Jefe/Gefe` > `Supervisor` > `Técnico`. Filtrado automático de visibilidad de usuarios según permisos.
- **Blindaje Anti-Debugging:**
  - Bloqueo global de menú contextual (click derecho).
  - Deshabilitación de atajos de inspección (F12, Ctrl+Shift+I/J/C).
  - Bloqueo de visualización de código fuente (Ctrl+U).
- **Protección de Diseño PWA:** Bloqueo sistemático de zoom manual (gestos táctiles, rueda, teclado) para mantener la integridad visual, con excepción inteligente activada para el Lightbox.
- **Integridad de Identidad:** Los usuarios pueden gestionar su nombre y contraseña, pero el nombre de usuario (`username`) permanece bloqueado para auditoría.

## 3. RENDIMIENTO Y OPTIMIZACIÓN MÓVIL
- **Gestión Tiered de Imágenes:** Estandarización de resoluciones (`SMALL`: 300px, `MEDIUM`: 800px, `LARGE`: 1600px) para minimizar el consumo de datos móviles.
- **Carga Diferida Sistemática:** Atributo `loading="lazy"` integrado en todo el catálogo y carruseles.
- **Carga Híbrida en Modales:** Implementación de carga diferida dentro de acordeones; las imágenes de cortes alternativos solo consumen ancho de banda al ser expandidas.
- **Adaptabilidad de Layout:** Uso de la API `visualViewport` para ajustar dinámicamente la altura de la aplicación (`--app-height`) y prevenir desajustes por el teclado virtual.

## 4. EXPERIENCIA TÉCNICA Y NAVEGACIÓN
- **Navegación Contextual Inteligente:** El sistema detecta el punto de entrada (Carrusel de Marcas vs. Búsqueda por Categoría) para que el botón "Volver" respete el flujo lógico del técnico.
- **Buscador de Alta Velocidad:** Filtrado en tiempo real con detección automática de marcas e integración visual de iconos de fabricantes.
- **Interfaz de Alta Densidad:** Rediseño compacto de tablas de gestión e Inbox para maximizar la visibilidad de datos técnicos en pantallas pequeñas.
- **Modo Oscuro Nativo:** Soporte integral con persistencia y cambio dinámico de branding (Logo_TemaOscuro.png).

## 5. GESTIÓN TÉCNICA DE CORTES (NUEVAS FEATURES)
- **Organización Automática por Utilidad:** Los cortes dentro del modal se ordenan dinámicamente según la cantidad de votos de "útil" de la comunidad.
- **Corte Recomendado Inteligente:** El sistema selecciona y destaca automáticamente el corte con mayor validación técnica como la opción principal.
- **Opciones de Cortes Múltiples:** Capacidad de visualizar hasta 3 variantes de corte (Bomba, Señal, Ignición) por cada vehículo.
- **Modales de Configuración Relay:** Vinculación directa a diagramas detallados y funciones de cada pin del relay según la configuración necesaria.
- **Crédito Colaborativo:** Identificación visible del técnico que aportó cada información de corte, fomentando la calidad y responsabilidad.

## 6. ADMINISTRACIÓN Y MEJORA CONTINUA
- **Módulo de Inbox Centralizado:** Sistema para que los técnicos reporten problemas o sugieran mejoras, con panel de administración para respuestas y resoluciones.
- **Dashboard de Actividad:** Registro en tiempo real de acciones (votos, logins, ediciones) para supervisión de calidad técnica.
- **Asistente Anti-Duplicado:** Flujo de registro en 3 etapas (`add_cortes.html`) que busca coincidencias antes de permitir nuevos registros, manteniendo la base de datos limpia.
- **Vídeo Guías Integradas:** Soporte nativo para incrustar guías de desarme de YouTube directamente en los detalles del vehículo.
